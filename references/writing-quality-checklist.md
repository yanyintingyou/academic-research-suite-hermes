# Writing Quality Checklist

Applied during draft self-review and report compilation. Flags machine-generated patterns without being adversarial.

## Anti-Patterns to Flag

1. **Overused AI-typical terms**
   - "delve", "tapestry", "landscape", "navigate", "robust", "multifaceted"
   - "In conclusion", "It is important to note", "As we have seen"
   - Replace with discipline-appropriate vocabulary

2. **Em-dash overuse**
   - More than 1 em-dash per paragraph is a yellow flag
   - More than 2 per paragraph is a red flag
   - Prefer parentheses or separate sentences

3. **Throat-clearing openers**
   - "In recent years, there has been growing interest in..."
   - "With the rapid development of..."
   - Start with the claim, not the context buildup

4. **Uniform paragraph lengths**
   - All paragraphs ~same word count = monotony
   - Vary: short punchy paragraphs for key claims, longer ones for exposition

5. **Monotonous sentence rhythm**
   - All sentences start with subject-verb
   - All sentences similar length
   - Vary: occasional leading prepositional phrase, varied clause structure

6. **Hedging inflation**
   - "might possibly suggest that it could be somewhat likely that..."
   - Match hedging to actual uncertainty; don't hedge every claim

## Checks by Section Type

| Section | Special Check |
|---------|--------------|
| Abstract | No citations; standalone meaning; keyword density |
| Introduction | Hook clarity; gap statement specificity; RQ placement |
| Method | Reproducibility; enough detail for replication |
| Results | Facts only, no interpretation (save for Discussion) |
| Discussion | Claim-evidence alignment; limitations honesty |
| References | Format consistency; DOI completeness |

## Style Calibration (Optional)

If 3+ past papers are provided:
1. Extract: sentence rhythm, vocabulary preferences, citation integration style
2. Apply as soft guide during drafting
3. Discipline conventions always take priority over personal style
